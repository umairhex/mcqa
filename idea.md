# MCQA - Product Concept Document

## Core Idea

### Problem
Educational assessments and knowledge checks suffer from multiple inefficiencies:
- Creating quizzes requires complex UI workflows or external tools
- Users need to manually track quiz progress and historical results
- Existing solutions either lack accessibility features or are overly complex
- Feedback on quiz performance is delayed or disconnected from the learning experience
- Quiz platforms lack flexibility for programmatic integration

### Solution
MCQA is a modern, **JSON-driven multiple-choice quiz engine** that empowers users to:
- Define quizzes using simple JSON format for maximum flexibility
- Instantly validate and load quiz questions
- Take quizzes with single-attempt, immediate feedback flow
- Automatically persist progress and results to local storage
- Access quiz history for quick reloading and retaking
- Integrate with AI systems for auto-generating quiz questions

MCQA bridges the gap between rigid quiz builders and technical solutions by providing a flexible, developer-friendly, yet user-accessible platform.

### Value
- **Speed**: Setup quizzes in minutes with simple JSON instead of hours with UI builders
- **Flexibility**: JSON format allows integration with any data source, AI systems, or content management system
- **Accessibility**: WCAG-compliant interface ensures all users can participate in assessments
- **Visibility**: Instant feedback and persistent history help users track learning progress in real-time
- **Developer Experience**: Clean API surface and typescript support for custom integrations

### User Outcome
After using MCQA, users can:
1. Create and deploy quizzes without leaving their workflow (JSON-first approach)
2. Immediately see their results with clear feedback on strengths and weaknesses
3. Review quiz history to track improvement over time
4. Generate quiz questions using AI integration
5. Access quizzes from any device with results seamlessly synced to local storage

## Key Features

### Feature Category 1: Dynamic Quiz Loading

**Purpose**
Enable users to rapidly create and deploy quizzes by providing quiz data in JSON format, eliminating the need for complex UI-based quiz builders.

**Capabilities**

- **JSON Input Interface**: Simple, clean interface for pasting or uploading quiz JSON
- **Strict Schema Validation**: Validates quiz structure against a strict schema, providing clear error messages for malformed data
- **Format Flexibility**: Supports quiz JSON from any source—handwritten, generated by AI, exported from external systems
- **Error Feedback**: Real-time validation feedback helps users fix issues before attempting the quiz
- **Instant Loading**: Quizzes load immediately after validation without page refresh

**User Benefit**
Reduces quiz creation time from hours to minutes. Content creators can focus on question quality rather than UI navigation. Non-technical users can use AI-generated quiz JSON directly.

---

### Feature Category 2: Strict Single-Attempt Quiz Flow

**Purpose**
Provide a focused, exam-like experience where users commit to their answers and receive immediate, undeniable feedback on correctness.

**Capabilities**

- **Answer Locking**: Once an answer is submitted, it cannot be changed—ensuring integrity of responses
- **Visual Correctness Indicators**: Immediate UI feedback showing correct answers in green, incorrect in red
- **Per-Question Feedback**: Users see whether their answer was correct immediately after submission
- **Timed Progression**: Users advance through questions one at a time, maintaining focus and reducing decision paralysis
- **Question Counter**: Shows current position in quiz (e.g., "Question 3 of 10") for context

**User Benefit**
Creates a reliable, test-like assessment experience. Users know exactly where they stand on each question. Prevents hedging or changing answers out of uncertainty, providing more authentic assessment data.

---

### Feature Category 3: Progress Persistence & Quiz History

**Purpose**
Automatically save user progress, quiz results, and historical attempts so users can track learning over time and quickly resume or retake quizzes.

**Capabilities**

- **Auto-Save to LocalStorage**: Quiz progress saved automatically on every answer submission
- **Quiz History Tracking**: Maintains up to 20 recent quizzes with timestamps, scores, and previews
- **Result Persistence**: Quiz results remain accessible even after browser restart
- **One-Click Reload**: Quickly reload a previous quiz from history without re-entering JSON
- **Cross-Session Continuity**: Progress syncs across browser sessions (within same device/browser)

**User Benefit**
Users never lose their progress. Quiz history provides a learning dashboard showing which topics they've practiced and their improvement over time. Reduces friction for retaking quizzes.

---

### Feature Category 4: Accessibility & Theme Support

**Purpose**
Ensure all users—regardless of ability or lighting conditions—can comfortably take quizzes and interact with the interface.

**Capabilities**

- **WCAG Compliance**: Full keyboard navigation support, ARIA labels, and screen reader compatibility
- **Dark/Light Mode**: System preference detection with manual override for user control
- **Responsive Design**: Optimized for desktop, tablet, and mobile devices
- **Clear Typography**: High-contrast text and readable font sizes
- **Inclusive Component Library**: Built with shadcn/ui for accessible, tested UI elements

**User Benefit**
Accessibility isn't an afterthought—it's built in. Users with disabilities can take quizzes independently. Dark mode reduces eye strain for long study sessions.

---

### Feature Category 5: AI-Ready Integration

**Purpose**
Position MCQA as a platform that can programmatically generate quiz questions from AI systems, enabling rapid, scalable quiz creation.

**Capabilities**

- **AI Prompt Framework**: Built-in prompt structure for passing to LLMs to generate quiz JSON
- **JSON Output Format**: AI-generated responses can be directly fed into MCQA
- **No Manual Conversion Needed**: Generated quizzes work immediately without reformatting
- **Extensible API Surface**: TypeScript types and clean utilities enable custom integrations

**User Benefit**
Users can generate quiz questions from any text, document, or topic using AI without manual quiz creation. Drastically speeds up content development for large-scale training programs.

## System Structure

### User Interface

**Main Workspace**: MCQA operates as a single-page application with three distinct phases rendered in a single container:

1. **Setup Phase**
   - Clean input area for pasting or uploading JSON
   - Real-time validation feedback
   - Error display with helpful guidance
   - Breadcrumb navigation showing current phase

2. **Quiz Phase**
   - Question display area (one question at a time)
   - Multiple choice option buttons
   - Visual indicators (locked/submitted state)
   - Progress counter (e.g., "3 of 10")
   - Navigation breadcrumb

3. **Results Phase**
   - Score summary (e.g., "8 out of 10 correct")
   - Per-question review with correct/incorrect indicators
   - Option to reload quiz from history or start over
   - Theme toggle available throughout

### Data Layer

**Quiz Schema** (in-memory and localStorage):
- Questions: Array of question objects (text, options, correct answer index)
- Options: Array of choice objects (text, metadata)
- User Answers: Mapping of question index to selected answer index
- Quiz Metadata: Title, timestamp, completion status
- Results: Score, percentage, time taken (if applicable)

**Storage Locations**:
- **Session State**: Current phase, active quiz data in React component state
- **LocalStorage**: Quiz history (up to 20 entries), completed results, user preferences (theme)

### Access Model

**Authentication**: None required—MCQA is a client-side, local-storage application. No server, no login.

**Access Scope**:
- Public access (anyone can visit and use)
- Quiz data is **user-provided** (not fetched from server)
- History is **browser-local** (not synchronized across devices)
- No user accounts or permissions system

**Privacy**: All data remains on the user's device. No data is sent to external servers (except for optional AI integration calls).

### Persistence

**Across Sessions**:
- Quiz history persists in browser localStorage
- User theme preference persists
- Results and scores persist indefinitely (until manually cleared)

**Across Devices**:
- No cross-device sync (by design—keeps infrastructure simple)
- Each device/browser maintains its own history

**Implementation**: Web Storage API (localStorage) for client-side persistence, ensuring instant load times and no server dependency.

## User Workflow

### Entry

Users arrive at MCQA and see:
1. A welcoming interface with setup instructions
2. A large text input area labeled "Paste your quiz JSON"
3. An "Optional: Load from History" section showing recent quizzes (if any exist)
4. Clear instructions on JSON format or a link to example quiz JSON

**First-Time Entry**: New users can either paste sample JSON or generate a quiz via AI.

**Returning Entry**: Users can click "Quick Load" on a recent quiz from history to immediately jump to taking the quiz.

### Creation

Users create a quiz by providing JSON in one of these ways:

1. **Manual Entry**: Type or paste quiz JSON directly into the input field
2. **Historical Reload**: Select a previous quiz from the history sidebar
3. **AI Generation**: Use AI prompt with MCQA's schema to generate quiz JSON
4. **External Export**: Export quiz JSON from another system and paste into MCQA

The JSON is instantly validated, and errors are clearly communicated with specific field guidance.

### Organization

Quizzes are organized by:

- **Temporal Order**: History displays recent quizzes first (newest to oldest)
- **Quiz Title**: Visible in history list for easy identification
- **Timestamp**: Shows when the quiz was last accessed
- **Preview Data**: Shows quiz count and quick access score (if completed)

No folder hierarchy or tagging—keeping the system simple and fast.

### Retrieval

Users retrieve quizzes through:

1. **Quick History Access**: One-click load of recent quizzes
2. **Search**: (Future feature) Filter history by quiz title or date
3. **Manual Input**: Paste quiz JSON again if not in history
4. **Bookmarking**: Save quiz JSON externally and paste when needed

### Reuse

Once quizzes are in history, users can:

1. **Retake**: Load a previous quiz and take it again (separate results recorded)
2. **Review Answers**: View their previous answers and correct answers
3. **Track Progress**: See score improvements across multiple attempts
4. **Share**: Export quiz JSON and share via email, LMS, or collaboration tools

## Product Positioning

### Category

**Interactive Assessment Platform** / **Developer-Friendly Quiz Engine**

More specifically: A lightweight, JSON-driven quiz tool positioned between rigid quiz builders (like Quizlet) and headless quiz APIs (like traditional backend solutions).

### Scope

**What MCQA Focuses On**:
- Single-attempt, immediate-feedback quiz experiences
- Client-side quiz execution (fast, no server required)
- Accessibility and inclusive design
- Developer flexibility through JSON format
- Simple, focused user interface

**What MCQA Intentionally Avoids**:
- User accounts, login systems, or server-side persistence (out of scope)
- Advanced quiz types (branching, adaptive paths, time-limited quizzes)
- Instructor dashboards or analytics backends
- Multimedia content (images, videos embedded in questions)
- Complex assessment workflows (surveys, polls, ranking questions)

### Primary Users

**Tier 1 (Core)**: 
- Educators creating quick knowledge checks
- Content creators building self-assessment tools
- Trainers deploying learning modules

**Tier 2 (Secondary)**:
- Students studying with peer-created quizzes
- Developers integrating quizzes into apps (via JSON API)
- Organizations auto-generating quizzes with AI

**Tier 3 (Long-tail)**:
- LMS platforms integrating MCQA as a quiz engine
- Corporate training programs
- Self-directed learners

### Core Value Proposition

**One-liner**: "The fastest way to create, deploy, and take multiple-choice quizzes—from JSON to results in seconds, with zero server setup required."

### Positioning Statement

MCQA is a modern assessment tool for anyone who wants to quickly create, share, and take multiple-choice quizzes without friction. Unlike traditional quiz builders (which require clicking through complex interfaces) or expensive LMS platforms (which require enterprise setup), MCQA uses simple JSON to define quizzes, instant feedback to reinforce learning, and local storage to persist progress—all in a blazingly fast, accessible, keyboard-friendly interface. Whether you're an educator testing student knowledge, a trainer deploying modules, or a developer building custom assessments, MCQA gets out of your way.

### Frontend Framework & Language

- **React 19** - Latest React for component-based UI and hooks
- **TypeScript 5** - Strict typing for reliability and DX

### Build & Development Tools

- **Vite 7** - Ultra-fast build tool and dev server
- **@vitejs/plugin-react** - Fast Refresh for hot module reloading
- **@tailwindcss/vite** - Tailwind CSS compiler as Vite plugin

### Styling & UI Components

- **Tailwind CSS 4** - Utility-first CSS framework for rapid, maintainable styling
- **shadcn/ui** - Accessible, pre-built component library (buttons, inputs, dialogs, etc.)

- **clsx + tailwind-merge** - Utility for merging Tailwind classes dynamically
- **next-themes** - Dark/light mode theme switching with system preference detection
- **tw-animate-css** - Animation utilities for Tailwind CSS

### Code Quality & Linting

- **ESLint 9** - JavaScript/TypeScript linting with React hooks plugin for best practices
- **Prettier** - Code formatter for consistent style
- **TypeScript strict mode** - Enforces strict type checking

### Package Management

- **pnpm** - Fast, disk-space efficient package manager (recommended)

- **Vercel** - Serverless deployment for static React apps (live at https://mcqa.vercel.app)

- **Web Storage API** - For localStorage-based quiz history and progress persistence
